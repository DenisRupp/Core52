Autoloading

Core52 provides completely automatic loading of Models and Libraries provided that the file name matches the class name. Note that file names are case sensitive on UNIX-type operating systems.

Autoload Sequence

When a new class is referenced, Core52′s autoloader looks for a matching file in the following directories, in order:

  1. app/models
  2. app/objects
  3. core/objects

Note that this allows for core objects to be overridden (versus extended) by simply putting the replacement in the app/objects directory.

Core52′s autoloader is built around the PHP spl_autoload_register() function, thereby allowing multiple cascading autoloaders to be registered. Thus, if the Core52 autoloader fails and multiple loaders are registered on the autoload stack, then Core52 will return FALSE to allow the next loader to take over.

Disabling Autoloading

To disable, simply call:

AutoClassLoader::Unregister();

To re-enable:

AutoClassLoader::Register();

Error Handling

If a class cannot be loaded, then an AutoClassLoaderException will be thrown. To disable this behavior, call:

AutoClassLoader::ThrowExceptions(FALSE);

To enable:

AutoClassLoader::ThrowExceptions(TRUE);
