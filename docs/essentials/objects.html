Objects

Usage

Perhaps confusingly named, Core52 Objects are essentially libraries that can encapsulate just about any functionality needed. Objects are ideal for functionality that is shared throughout your application.

Store Objects in your app/objects directory.

Naming

  • Don't use weird characters in your model names. Stick with letters, numbers, and underscores. Don't start a model name with a number.
  • Object file names are case sensitive (on UNIX) and must match the Object class name.

Pattern

<?php

class Import {

}

There is no base Object class, and you have full, free reign designing your Objects. You can also drop in third-party classes here–just make sure the naming rules above are followed so that autoloading doesn’t break.

It is possible to include multiple classes in one object file, however if these classes are used outside of your primary Object class then you will need to manually include() the object before referencing any of those classes. It is generally best to limit objects to one class per file.

Core Objects

Core52 provides numerous objects in the core/objects directory. To learn about the functionality that is available, see the Core52 Libraries section of the documentation.

Note that if an app Object is the same name as a Core52 Object, the app Object will override the Core52 Object. See “Overriding Core52″ in the Hacking section.
